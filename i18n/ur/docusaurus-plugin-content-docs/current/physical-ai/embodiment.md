## 1. مجسم ذہانت کیا ہے؟

**مجسم ذہانت** یہ اصول ہے کہ ذہین رویہ ایک ایجنٹ کے دماغ (یا کنٹرولر)، جسم (مورفولوجی)، اور ماحول کے درمیان متحرک تعامل سے ابھرتا ہے۔ روایتی AI سسٹمز کے برعکس جو تجریدی علامتوں کو تنہائی میں پروسیس کرتے ہیں، مجسم AI سسٹمز کو طبعی دنیا کا مقابلہ کرنا پڑتا ہے—اس کا شور، تاخیر، رکاوٹیں، اور مواقع۔

> "ذہانت کو اس سیاق و سباق کو سمجھے بغیر نہیں سمجھا جا سکتا جس میں یہ کام کرتی ہے: جسم اور ماحول۔"
> — Rolf Pfeifer & Josh Bongard, *How the Body Shapes the Way We Think*

### بنیادی بصیرت

سوچیں کہ آپ گیند کو کیسے پکڑتے ہیں۔ آپ شعوری طور پر پیرابولک رفتار کا حساب نہیں لگاتے یا تفریقی مساوات حل نہیں کرتے۔ اس کے بجائے، آپ کا جسم ایک سادہ حکمت عملی استعمال کرتا ہے: حرکت کرتے رہیں تاکہ گیند آپ کے بصری میدان میں ایک سیدھی لکیر میں حرکت کرتی ہوئی دکھائی دے۔ اسے **نگاہ کا ہیورسٹک (gaze heuristic)** کہا جاتا ہے—ایک کمپیوٹیشنل طور پر سستا حل جو بالکل اسی لیے کام کرتا ہے *کیونکہ* آپ کے پاس ایک جسم ہے جو حرکت کر سکتا ہے۔

یہ مثال مجسم ذہانت کی ایک بنیادی بصیرت کی وضاحت کرتی ہے: **جسم صرف دماغ کے لیے ایک برتن نہیں ہے—یہ ادراکی نظام کا ایک لازمی حصہ ہے**۔

### مجسم بمقابلہ غیر مجسم AI

| پہلو | غیر مجسم (Disembodied) AI | مجسم (Embodied) AI |
|--------|----------------|-------------|
| **ماحول** | نقلی یا ڈیجیٹل | طبعی دنیا |
| **فیڈبیک** | فوری، کامل | تاخیر والا، شور والا |
| **وقت** | اکثر تجریدی | حقیقی وقت کی رکاوٹیں |
| **جسم** | کوئی نہیں یا غیر متعلقہ | حساب کتاب کا مرکز |
| **سیکھنا** | ڈیٹا سے | تعامل سے |
| **مثالیں** | ChatGPT، شطرنج کے انجن | روبوٹس، خود مختار گاڑیاں |

## 2. تاریخی سیاق و سباق: علامتی AI سے مجسم تک

### علامتی AI کا دور (1956-1980 کی دہائی)

ابتدائی AI تحقیق، جس کی مثال **فزیکل سمبل سسٹم ہائپوتھیسس** (نیویل اور سائمن، 1976) ہے، نے فرض کیا کہ ذہانت رسمی اصولوں کے مطابق تجریدی علامتوں کے جوڑ توڑ کے ذریعے حاصل کی جا سکتی ہے۔ یہ نقطہ نظر ماہر نظام اور منطقی استدلال کے انجنوں کی طرف لے گیا۔

تاہم، علامتی AI نے جدوجہد کی:
- **فریم کا مسئلہ**: اعمال سے تمام متعلقہ تبدیلیوں کی وضاحت کرنا
- **علامت کی بنیاد کا مسئلہ**: علامتوں کو حقیقی دنیا کے معنی سے جوڑنا
- **نازکی (Brittleness)**: غیر متوقع حالات کا سامنا کرنے پر ناکامی۔

### رویے کا موڑ (1980-1990 کی دہائی)

Rodney Brooks کے بااثر مقالے "Intelligence Without Representation" (1991) نے علامتی پیراڈائم کو چیلنج کیا۔ بروکس نے دلیل دی کہ:

1. دنیا خود اپنا بہترین ماڈل ہے
2. پیچیدہ رویہ ماحول کے ساتھ تعامل کرنے والے سادہ اصولوں سے ابھر سکتا ہے
3. نمائندگی اور استدلال اکثر غیر ضروری ہوتے ہیں

ان کے **subsumption architecture** نے دکھایا کہ کیڑے مکوڑوں جیسے رویے مرکزی منصوبہ بندی یا دنیا کے ماڈل کے بغیر حاصل کیے جا سکتے ہیں۔

### مجسم ادراک کی تحریک

رویے والے روبوٹکس پر تعمیر کرتے ہوئے، علمی سائنس کے محققین نے **مجسم ادراک کا نظریہ** تیار کیا، جو یہ مانتا ہے کہ:

- ادراک جسم سے تشکیل پاتا ہے
- ذہن دماغ سے آگے بڑھتا ہے
- سوچ سینسروموٹر تجربے میں پیوست ہے

اہم شخصیات میں شامل ہیں:
- **جارج لیکوف اور مارک جانسن**: جسمانی تجربے میں پیوست تصوراتی استعارے
- **اینڈی کلارک**: توسیعی ذہن کا مفروضہ
- **ایستھر تھیلن**: ترقی کا متحرک نظام نقطہ نظر

## 3. سینسروموٹر لوپ (The Sensorimotor Loop)

مجسم ذہانت کے مرکز میں **سینسروموٹر لوپ** ہے—حس، پروسیسنگ، اور عمل کا مسلسل چکر جو ذہین رویے کی بنیاد بناتا ہے۔

### لوپ کی ساخت

```
┌─────────────────────────────────────────────────────┐
│                    ماحول (Environment)               │
└──────────────┬─────────────────────┬────────────────┘
               │                     │
         ادراک (Perception)    عمل کا اثر (Action Effect)
               │                     │
               ▼                     │
        ┌──────────┐                 │
        │  سینسرز  │                 │
        └────┬─────┘                 │
             │                       │
      حسی ڈیٹا (Sensory Data)         │
             │                       │
             ▼                       │
     ┌───────────────┐               │
     │   کنٹرولر     │               │
     │    (دماغ)     │               │
     └───────┬───────┘               │
             │                       │
      موٹر کمانڈز                    │
             │                       │
             ▼                       │
       ┌───────────┐                 │
       │ ایکچیوٹرز │─────────────────┘
       └───────────┘
```

### کلیدی خصوصیات

1. **کلوزڈ لوپ کنٹرول**: اعمال ماحول کو متاثر کرتے ہیں، جو سینسر کی ریڈنگ کو تبدیل کرتا ہے، جو مستقبل کے اعمال پر اثر انداز ہوتا ہے۔

2. **حقیقی وقت کی رکاوٹیں**: ماحولیاتی تبدیلیوں کا جواب دینے کے لیے لوپ کو کافی تیزی سے چلنا چاہیے۔

3. **شور اور غیر یقینی صورتحال**: سینسر اور ایکچیوٹرز دونوں نامکمل ہیں۔

4. **جوڑا (Coupling)**: ایجنٹ اور ماحول متحرک طور پر جڑے ہوئے ہیں—کسی کو بھی تنہائی میں نہیں سمجھا جا سکتا۔

### مثال: غیر فعال متحرک واکر (Passive Dynamic Walker)

غیر فعال متحرک واکر جسم اور ماحول کے جوڑے کی طاقت کو ظاہر کرتا ہے۔ یہ سادہ میکانزم بغیر کسی موٹرز، سینسرز، یا کنٹرولرز کے ڈھلوان پر چلتا ہے—صرف کشش ثقل اور ٹانگوں کی ہوشیار جیومیٹری سے۔

```python
"""
غیر فعال متحرک چلنے کی طبیعیات کا آسان تخروپن۔
یہ ظاہر کرتا ہے کہ کس طرح جسمانی ساخت رویے میں حصہ ڈالتی ہے۔
"""

import numpy as np

class PassiveWalkerLeg:
    """غیر فعال متحرک واکر ٹانگ کا ایک سادہ ماڈل۔"""

    def __init__(self, length: float = 1.0, mass: float = 1.0):
        self.length = length
        self.mass = mass
        self.angle = 0.0  # عمودی سے زاویہ
        self.angular_velocity = 0.0

    def compute_acceleration(self, slope_angle: float, gravity: float = 9.81) -> float:
        """
        ڈھلوان پر کشش ثقل کی وجہ سے کونیی سرعت کا حساب لگائیں۔

        کلیدی بصیرت: ٹانگ کی جیومیٹری قدرتی طور پر فعال کنٹرول کے بغیر
        چلنے کا رویہ پیدا کرتی ہے۔
        """
        # آسان پینڈولم ڈائنامکس
        # حقیقت میں، اس میں حرکت کی مکمل مساوات شامل ہیں
        return (gravity / self.length) * np.sin(self.angle - slope_angle)

    def step(self, dt: float, slope_angle: float) -> None:
        """تخروپن کو ایک ٹائم سٹیپ آگے بڑھائیں۔"""
        acceleration = self.compute_acceleration(slope_angle)
        self.angular_velocity += acceleration * dt
        self.angle += self.angular_velocity * dt

# واکر الگورتھم کے ذریعے نہیں بلکہ طبیعیات کے ذریعے اپنی چال کا "حساب" لگاتا ہے
walker = PassiveWalkerLeg(length=1.0)
slope = 0.05  # ریڈین میں ہلکی ڈھلوان

print("Passive Dynamic Walking Demonstration")
print("=" * 40)
print("نوٹ: چلنا طبیعیات سے ابھرتا ہے، کنٹرول سے نہیں!")
print(f"ٹانگ کی لمبائی: {walker.length}m, ڈھلوان: {np.degrees(slope):.1f}°")
```

**آؤٹ پٹ:**
```
Passive Dynamic Walking Demonstration
========================================
نوٹ: چلنا طبیعیات سے ابھرتا ہے، کنٹرول سے نہیں!
ٹانگ کی لمبائی: 1.0m, ڈھلوان: 2.9°
```

یہ مثال دکھاتی ہے کہ کس طرح ذہین نظر آنے والا رویہ (چلنا) ماحول کے ساتھ تعامل کرنے والی جسمانی ساخت سے ابھر سکتا ہے، بغیر کسی روایتی معنوں میں حساب کتاب کے۔

## 4. مورفولوجیکل کمپیوٹیشن (Morphological Computation)

**مورفولوجیکل کمپیوٹیشن** سے مراد وہ طریقہ ہے جس میں ایجنٹ کے جسم کی شکل اور طبعی خصوصیات ذہین رویے کے لیے درکار کمپیوٹیشنل کام کا حصہ انجام دیتی ہیں۔

### تصور

روایتی روبوٹکس جسم کو ایک مسئلے کے طور پر دیکھتا ہے جس پر قابو پانا ہے—شور، غلطی، اور رکاوٹوں کا ذریعہ۔ مورفولوجیکل کمپیوٹیشن اس نقطہ نظر کو پلٹ دیتا ہے: جسم ایک **کمپیوٹیشنل وسیلہ** ہے۔

### مورفولوجیکل کمپیوٹیشن کی مثالیں

#### 1. انسانی ہاتھ

انسانی ہاتھ میں آزادی کے 27 درجے (degrees of freedom) اور پیچیدہ پٹھے ہوتے ہیں، لیکن ہم شعوری طور پر ہر جوڑ کو کنٹرول نہیں کرتے۔ ہاتھ کی **لچک (compliance)**—غیر فعال طور پر شکل بدلنے کی اس کی صلاحیت—پکڑنے کے بہت سے مسائل کو خود بخود حل کرتی ہے۔

```python
"""
یہ ظاہر کرنا کہ نرم/لچکدار جسم کنٹرول کو کس طرح آسان بناتے ہیں۔
"""

class RigidGripper:
    """روایتی سخت گریپر جس کے لیے درست پوزیشننگ کی ضرورت ہوتی ہے۔"""

    def grasp(self, object_position: float, object_size: float) -> bool:
        gripper_position = 0.0
        tolerance = 0.01  # بہت درست ہونا ضروری ہے

        error = abs(gripper_position - object_position)
        if error > tolerance:
            return False  # ناکام - کافی درست نہیں
        return True

class SoftGripper:
    """لچکدار گریپر جو اشیاء کے مطابق ڈھل جاتا ہے۔"""

    def grasp(self, object_position: float, object_size: float) -> bool:
        # نرم مواد چیز کے گرد شکل بدل لیتا ہے
        # لچک کی وجہ سے بہت زیادہ رواداری
        tolerance = 0.5  # پوزیشننگ کی اہم غلطی کو سنبھال سکتا ہے

        gripper_position = 0.0
        error = abs(gripper_position - object_position)
        if error > tolerance:
            return False
        return True  # لچک باقی کام سنبھال لیتی ہے

# پوزیشننگ کی غیر یقینی صورتحال کے ساتھ کامیابی کی شرح کا موازنہ کریں
import random

rigid = RigidGripper()
soft = SoftGripper()

rigid_successes = 0
soft_successes = 0
trials = 100

for _ in range(trials):
    # بے ترتیب پوزیشننگ غلطی (حقیقی دنیا کی غیر یقینی صورتحال کی نقل)
    position_error = random.gauss(0, 0.1)

    if rigid.grasp(position_error, 0.05):
        rigid_successes += 1
    if soft.grasp(position_error, 0.05):
        soft_successes += 1

print(f"سخت گریپر کی کامیابی کی شرح: {rigid_successes}%")
print(f"نرم گریپر کی کامیابی کی شرح: {soft_successes}%")
print("\nنرم گریپر کی مورفولوجی کنٹرول کے بوجھ کو کم کرتی ہے!")
```

**آؤٹ پٹ:**
```
سخت گریپر کی کامیابی کی شرح: 8%
نرم گریپر کی کامیابی کی شرح: 100%

نرم گریپر کی مورفولوجی کنٹرول کے بوجھ کو کم کرتی ہے!
```

#### 2. کیڑے کے پنکھ

کیڑے کے پنکھ ہر مقام پر سختی سے کنٹرول نہیں کیے جاتے۔ ان کی لچکدار ساخت خود بخود ایروڈائینامک قوتوں کے مطابق ہو جاتی ہے، جو کم سے کم اعصابی کنٹرول کے ساتھ مستحکم پرواز کو ممکن بناتی ہے۔

#### 3. مونچھیں اور اینٹینا

چوہے کی مونچھیں اور کیڑے کے اینٹینا مکینیکل سینسر ہیں جو اپنی جسمانی حرکیات کے ذریعے معلومات کو پہلے سے پروسیس کرتے ہیں، اعصابی نظام پر کمپیوٹیشنل بوجھ کم کرتے ہیں۔

## 5. فطرت میں مجسم ادراک

فطرت مجسم ذہانت کی بے شمار مثالیں فراہم کرتی ہے جو روبوٹکس کی تحقیق کو مطلع کرتی ہیں۔

### کیس اسٹڈی 1: چیونٹی کی نیویگیشن

صحرائی چیونٹیاں (*Cataglyphis*) خوراک تلاش کرنے اور براہ راست اپنے گھونسلے میں واپس آنے کے لیے وسیع فاصلے طے کرتی ہیں۔ وہ تفصیلی نقشے نہیں بناتیں۔ اس کے بجائے، وہ استعمال کرتی ہیں:

1. **راستہ انضمام**: قدموں کی گنتی اور سمت کا سراغ لگانا
2. **بصری سنیپ شاٹس**: سادہ تصویر کا موازنہ
3. **جسمانی میکانکس**: ٹانگوں کی ساخت قدموں کی گنتی کے لیے موزوں ہے

چیونٹی کا جسم اس کے نیویگیشن "کمپیوٹر" کا حصہ ہے۔

### کیس اسٹڈی 2: آکٹوپس بازو کا کنٹرول

آکٹوپس کے آٹھ بازو ہیں جن میں آزادی کے عملی طور پر لامحدود درجے ہیں۔ اسے مرکزی طور پر کنٹرول کرنے کے لیے بہت زیادہ کمپیوٹیشنل طاقت کی ضرورت ہوگی۔ اس کے بجائے:

- ہر بازو میں نمایاں مقامی ذہانت ہوتی ہے (2/3 نیوران بازوؤں میں ہوتے ہیں)
- بازو نیم خود مختار طور پر کام انجام دے سکتے ہیں
- مرکزی دماغ اہداف طے کرتا ہے؛ بازو تفصیلات کا پتہ لگاتے ہیں

یہ **تقسیم شدہ کنٹرول** پیچیدہ روبوٹ مورفولوجیز کے لیے ایک کلیدی اصول ہے۔

## 6. روبوٹکس کے لیے مضمرات

### مجسم ذہانت کے ڈیزائن اصول

#### اصول 1: مقام کے لیے ڈیزائن (Design for the Niche)

عام مقصد والے روبوٹ بنانے کے بجائے، مخصوص ماحول کے لیے ڈیزائن کریں:

#### اصول 2: ماحول کا استحصال کریں

ماحول کی خصوصیات کو اپنے حل کے حصے کے طور پر استعمال کریں:
- لوکلائزیشن کے لیے دیواریں
- حرکت کے لیے کشش ثقل
- نیویگیشن کے لیے زمینی خصوصیات

#### اصول 3: جسم کو کام سے ملائیں

جسم کو اس طرح ڈیزائن کیا جانا چاہیے کہ کنٹرول کے مسئلے کو آسان بنایا جا سکے:
- غیر ساختہ ماحول میں ہیرا پھیری کے لیے نرم جسم
- موثر حرکت کے لیے کم ایکچیوٹڈ ڈیزائن
- محفوظ انسانی تعامل کے لیے لچکدار جوڑ

#### اصول 4: شور اور نامکمل پن کو گلے لگائیں

طبعی خامیوں سے لڑنے کے بجائے، ایسے سسٹم ڈیزائن کریں جو ان کے ساتھ کام کریں:

```python
"""
مضبوط کنٹرولر جو سینسر کے شور کو گلے لگاتا ہے۔
"""

class RobustController:
    """شور والے سینسر کے ساتھ کام کرنے کے لیے ڈیزائن کیا گیا کنٹرولر۔"""

    def __init__(self, noise_tolerance: float = 0.2):
        self.noise_tolerance = noise_tolerance
        self.history = []

    def process_sensor(self, reading: float) -> float:
        """شور کو سنبھالنے کے لیے متعدد حکمت عملیوں کا استعمال کریں۔"""
        self.history.append(reading)

        # صرف حالیہ ریڈنگز رکھیں
        if len(self.history) > 10:
            self.history.pop(0)

        # سادہ لیکن مؤثر: میڈین کا استعمال کریں (آؤٹ لائیرز کے خلاف مضبوط)
        return sorted(self.history)[len(self.history) // 2]
```

## 7. فوائد اور حدود

### مجسم AI کے فوائد

1. **کم کمپیوٹیشنل ضروریات**: جسم حساب کتاب کا حصہ سنبھالتا ہے
2. **قدرتی مضبوطی**: طبعی نظام اکثر خوبصورتی سے خراب ہوتے ہیں
3. **توانائی کی کارکردگی**: غیر فعال حرکیات کا استحصال توانائی بچاتا ہے
4. **بدیہی ڈیزائن**: فطرت سے الہام حاصل کیا جا سکتا ہے
5. **اندرونی بنیاد**: معنی جسمانی تعامل سے آتا ہے

### حدود اور چیلنجز

1. **تجزیہ کی مشکل**: رویہ پیچیدہ تعاملات سے ابھرتا ہے
2. **ڈیزائن کی پیچیدگی**: جسم اور کنٹرولر کو ایک ساتھ بہتر بنانا مشکل ہے
3. **جسموں میں منتقلی**: مہارتیں مختلف مورفولوجیز کے درمیان آسانی سے منتقل نہیں ہوتیں
4. **تخروپن کے چیلنجز**: جسمانی تعاملات کی درست نقالی کرنا مشکل ہے
5. **مرمت اور ترمیم**: سافٹ ویئر کے مقابلے میں طبعی سسٹمز کو اپ ڈیٹ کرنا مشکل ہے
